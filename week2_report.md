## 一、本周完成内容

### 1. 分词与词表构建（`scripts/vocab.py`）
根据第一周 EDA 结论（模板化语料、词汇量小），确定三种语言的分词方案：

| 语言 | 分词方案 | 词表规模 | val/test 未登录词率 |
|------|----------|----------|---------------------|
| 中文（源端） | jieba 词级 + 术语整词 | 135 | 0% |
| 日语（目标端） | 字符级 | 195 | 0% |
| 印尼语（目标端） | 小写 + 空格 | 185 | 0% |

- 把第一周术语表的 36 个术语加入 jieba 用户词典，让`故障恢复`等术语保持整词，中文词表从 149 收敛到 135。
- 日语只有 191 个字符，字符级分词规模可控，且无需安装日语分词器。
- 词表仅用训练集构建，避免数据泄漏；编码解码往返自检通过。

### 2. 特征工程（`scripts/features.py`）
按任务书要求提取四类特征：

**技术术语特征（TF-IDF）**：提取候选术语 107 个，其中 35 个在第一周术语表中。按 topic 提取的关键术语与领域相关（如数据库运维对应`索引优化`、`备份恢复`、`读写分离`）。产出 `term_features.csv`，供第三周评估术语翻译准确率使用。

**语义向量特征（Word2Vec）**：三语各训练 128 维词向量，维度与模型 d_model 一致，用于初始化 embedding 层。近邻抽查显示，术语的近邻反映的是模板槽位而非语义（`快照`、`带宽`、`重启`相似度均在 0.97 以上，因为它们出现在相同模板的相同位置），这符合模板语料的特点。

**句式结构特征（模板骨架）**：把术语替换为占位符后，训练集 3,592 句只有 150 种句式骨架。各 topic 骨架数与第一周重复率负相关（数据库运维 10 种，配置类约 120 种），印证第一周"模板单一"的判断。产出 `template_features.csv`。

**上下文关联特征**：句级语料无跨句上下文，用 topic 作为上下文特征，为每个 topic 建立前缀 token 拼在源句句首。

### 3. 基线 Transformer 模型（`scripts/model.py`）
基于 PyTorch `nn.Transformer` 搭建 encoder-decoder。参数依据数据规模设定（数据量小，模型需小以防过拟合）：

| 参数 | 取值 |
|------|------|
| d_model | 128 |
| 注意力头数 | 4 |
| encoder / decoder 层数 | 2 / 2 |
| FFN 维度 | 512 |
| dropout | 0.1 |
| 总参数量 | 0.99M |

d_model 取 128 是为了与 Word2Vec 维度对齐，可用预训练词向量初始化 embedding。

### 4. 训练与初步验证（`scripts/train.py`）
训练配置：Adam（lr=3e-4）、teacher forcing、label smoothing 0.1、梯度裁剪 1.0、batch 64、早停 patience 8、固定随机种子。每轮监控 train/val loss。

| 方向 | 轮数 | 最优 val loss | val BLEU | 单轮耗时（CPU） |
|------|------|---------------|----------|-----------------|
| zh→ja | 60 | 0.8785 | 100.0（字符级） | ~73s |
| zh→id | 60 | 0.8672 | 100.0（词级，忽略大小写） | ~31s |

训练过程无异常（无 NaN，train/val loss 同步平稳下降，无过拟合），曲线见 `logs/train_curve_ja.png`、`logs/train_curve_id.png`。

## 二、关键发现：val BLEU=100 的解读

BLEU 满分不代表达到专业翻译水平。为定位真实含义，做了三项检查：

| 检查项 | 结果 |
|--------|------|
| val 原句与 train 完全重复 | 0 / 449 |
| val 模板骨架在 train 中出现过 | 449 / 449 |
| val 的（模板 + 术语组合）在 train 中出现过 | 0 / 449 |

结论分两面：
- 每条 val 句子都是训练中未见过的术语组合，模型全部翻对，说明模型学会了"模板框架翻译 + 术语填充"的组合能力，而非记忆句子。
- val 中所有模板骨架训练时都见过，模型对新句式的泛化能力没有被测试。150 种模板是这份数据集的上限，BLEU 在当前验证集上已无区分度。

## 三、遗留问题与下周建议

1. **评估基准需升级**：当前随机划分的验证集测不出句式泛化。建议按模板骨架重新划分，留出部分骨架不参与训练。
2. **补充术语翻译准确率指标**：BLEU 已饱和，且项目核心是术语准确性。可基于术语表逐术语核对译文。
3. **引入真实文档句子做人工评测**：检验模型在自然句式上的表现。
4. **验证 Word2Vec 初始化的作用**：本周默认开启，其实际贡献待第三周做对比实验确认。
